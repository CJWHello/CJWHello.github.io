# 多模态大模型算法学习路线

> 目标岗位：多模态大模型算法工程师  
> 背景假设：已会 PyTorch，熟悉 LoRA，可直接进入论文复现、模型训练和工程优化。  
> 方向覆盖：Video Generation、Vision Language Model、Diffusion、RLHF、World Model。  
> 时间规划：1 年，L1-L5 五个阶段，每月产出一个 GitHub 项目，最终形成可投递 OpenAI、DeepMind、字节 Seed、腾讯混元、阿里通义、快手可灵、MiniMax 等团队的项目组合。

## 总体策略

这条路线不是“泛读很多论文”，而是“用一年的时间做出一组可面试、可展示、可扩展的研究工程项目”。每个阶段都要求同时推进三件事：

1. 理论主线：理解模型为什么这样设计。
2. 复现主线：从最小实现到可训练系统。
3. 工程主线：从单卡 notebook 走到多卡训练、推理加速、评测和部署。

最终作品集至少包括：

| 方向 | 最终项目 | 面试价值 |
|---|---|---|
| VLM | Mini-VLM / LLaVA-style 多模态助手 | 证明你理解视觉编码器、投影器、LLM 对齐、SFT、多模态评测 |
| Diffusion | Text-to-Image Diffusion from Scratch + LoRA 微调 | 证明你理解 DDPM、DDIM、CFG、VAE、U-Net/DiT、采样器 |
| Video Diffusion | Mini Video Diffusion / DiT Video Generator | 证明你理解时空建模、3D attention、latent video diffusion、数据管线 |
| Agent | 多模态 Agent：看图/视频、调用工具、写报告 | 证明你能把 VLM、检索、工具调用和评测系统串起来 |
| RLHF | Preference/RLHF Lab：DPO/ORPO/GRPO/RM/PRM | 证明你理解后训练、偏好优化、奖励建模、可验证奖励 |
| World Model | Mini World Model：Dreamer/Genie-style Video Prediction | 证明你理解 latent dynamics、video prediction、action-conditioned generation |

## 最终项目组合设计

### 1. VLM 项目：`mini-vlm-blip2-llava`

目标：做一个可以看图回答、OCR、简单 grounding 的小型 VLM。

必须实现：

- CLIP/SigLIP 或 ViT encoder。
- Linear projector 或 Q-Former。
- LLM 接入，优先选择 Qwen/Llama 系小模型。
- 两阶段训练：image-text alignment、visual instruction tuning。
- 支持 LoRA。
- 评测：VQA、caption、OCR、MME/MMBench 子集或自建小 benchmark。
- README 中写清楚和 BLIP2、Flamingo、LLaVA、Qwen2-VL、InternVL、Molmo 的关系。

### 2. Diffusion 项目：`stable-diffusion-lora-lab`

目标：证明你不仅会调用 diffusers，还理解核心训练和采样。

必须实现：

- DDPM/DDIM from scratch。
- Stable Diffusion pipeline 拆解。
- LoRA fine-tuning。
- scheduler 对比。
- CFG scale、steps、resolution、seed 的 ablation。
- 失败案例：过拟合、prompt 不敏感、颜色/结构崩坏。

### 3. Video Diffusion 项目：`mini-video-diffusion`

目标：做一个缩小版 video diffusion，能讲清楚视频生成比图像生成难在哪里。

必须实现：

- latent video dataset loader。
- temporal attention 或 3D U-Net/DiT block。
- 8-16 帧 text/image-to-video demo。
- FVD/LPIPS 或至少 qualitative temporal consistency 分析。
- 对 CogVideoX、HunyuanVideo、Wan2.1、Open-Sora、MovieGen、SeedVideo 的公开技术对比。

### 4. Agent 项目：`modern-vlm-grounding-agent`

目标：做一个多模态研究助理，可以读取图像/视频帧、调用工具、输出结构化报告。

必须实现：

- 图片/视频输入。
- OCR、caption、grounding 或 frame retrieval。
- 工具调用：检索、本地文件、Python 分析。
- 评测集：10-50 个自建任务，记录成功率。
- 部署：vLLM 或 SGLang。

### 5. RLHF 项目：`preference-optimization-lab`

目标：把 PPO、DPO、ORPO、SimPO、GRPO、RFT、RLVR、RM、PRM 放进一个统一实验框架。

必须实现：

- SFT baseline。
- Reward Model pairwise training。
- DPO/ORPO/SimPO 对比。
- GRPO + verifiable reward。
- PRM step-level reward。
- RFT 实验报告：明确采用 Rejection Sampling Fine-Tuning 或 Reinforcement Fine-Tuning 语义。
- 指标：win rate、reward margin、KL、length bias、failure cases。

### 6. World Model 项目：`mini-world-model`

目标：从 video prediction 走到可交互 world model toy demo。

必须实现：

- MovingMNIST/Atari/robot video 数据之一。
- latent encoder。
- dynamics model。
- next-frame / multi-step rollout。
- action-conditioned 或 latent-action prediction。
- Dreamer 与 Genie/Genie2 思路对比。
- 失败分析：long horizon drift、object permanence、action ambiguity。

## 最后建议

如果你的目标是投递一线多模态团队，最重要的不是“读过多少 SOTA”，而是每个方向至少有一个能被追问 30 分钟的项目。面试官真正看重的是：

1. 你能不能把论文缩小成可复现实验。
2. 你能不能解释训练失败、显存爆炸、指标不涨的原因。
3. 你能不能把 VLM、Diffusion、Video、RLHF、World Model 放到同一个技术地图里。
4. 你有没有工程品味：清晰代码、可复现脚本、评测、ablation、失败分析。
5. 你是否知道哪些模型能复现，哪些只能做公开资料复盘，哪些需要用更小 proxy task 验证思想。

一年后，你应该能拿着 6 个核心项目和 12 个 GitHub 月度项目，清楚地讲出自己对多模态大模型的系统理解：从图文对齐，到图像生成，到视频生成，到偏好优化，到可交互世界模型。
